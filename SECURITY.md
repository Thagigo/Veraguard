# Security Policy

## Supported Versions

Only the latest `main` branch is currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**VeraGuard takes security seriously.** We treat all security vulnerabilities as critical.

If you discover a security vulnerability within VeraGuard, please report it privately:

1.  **Do NOT open a public issue.**
2.  Email our security team at `security@veraguard.io`.
3.  Provide a detailed description of the vulnerability and steps to reproduce.

### Response Timeline

-   **Acknowledgement**: Within 24 hours.
-   **Assessment**: Within 48 hours.
-   **Fix**: Critical vulnerabilities are patched within 72 hours.

## Medical-Grade Security Standards

VeraGuard adheres to strict security protocols:
-   **No Unchecked Code**: All code must pass CI/CD pipelines.
-   **Signature Verification**: Core logic is mathematically verified against known threat vectors.
-   **Zero Trust**: We assume all inputs are potentially malicious until proven otherwise.
