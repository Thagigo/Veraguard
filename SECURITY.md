# Security Policy

## Supported Versions
We actively provide security updates for the following versions of VeraGuard:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability
We take the security of VeraGuard seriously. If you believe you have found a security vulnerability, please do not open a public issue. Instead, follow these steps:

1. **Private Disclosure:** Report the vulnerability by [filling out the security advisory form](https://github.com/YOUR_USERNAME/YOUR_REPO/security/advisories/new) or emailing security@veraguard.io.
2. **Details to Include:** Provide a precise description, the potential impact, and step-by-step instructions to reproduce the findings.
3. **Response Timeline:** You will receive an initial acknowledgement within 48 hours. We aim to provide a full triage report within one week.

Please keep all reports confidential until we have officially made a public announcement.
